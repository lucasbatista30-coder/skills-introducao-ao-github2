### E agora?

Se quiser criar um README de perfil, use as instruções rápidas abaixo ou siga as instruções no artigo [Gerenciando seu README de perfil](https://docs.github.com/account-and-profile/setting-up-and-managing-your-github-profile/customizing-your-profile/managing-your-profile-readme).

1. Crie um novo repositório público com um nome igual ao seu nome de usuário do GitHub.
2. Crie um arquivo chamado `README.md` na raiz do repositório. "Raiz" significa não estar dentro de nenhuma pasta do seu repositório.
3. Edite o conteúdo do arquivo `README.md`.
4. Se você criou uma nova branch para o seu arquivo, abra e faça o merge de um pull request nessa branch.
5. Por fim, adoraríamos saber o que você achou deste curso [em nosso fórum de discussão](https://github.com/orgs/skills/discussions/categories/introduction-to-github).

Confira estes recursos para aprender mais ou se envolver:

- Você é estudante? Veja o [Student Developer Pack](https://education.github.com/pack).
- [Faça outro curso do GitHub Skills](https://github.com/skills).
- [Leia a documentação de primeiros passos do GitHub](https://docs.github.com/pt/get-started).
- Para encontrar projetos para contribuir, acesse o [GitHub Explore](https://github.com/explore).
